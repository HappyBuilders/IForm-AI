# IForm-AI 快速问题索引

> 用途：当用户通过自然语言提问时，优先读取本索引，直接将问题映射到最相关的参考文档，避免递归扫描整个 `references/`。

## 使用规则

1. 先从用户问题中提取 2~5 个关键词：
   - 业务词：审批意见、节点、回写、控件、校验、提交、打印、权限、接口、同步
   - 异常词：不生效、没回写、没显示、报错、异常、不一致、无法提交
   - 对象词：主表、子表、明细、字段、表单、流程、移动端、PC端
2. 先查本索引，命中后直接读取对应 1~3 篇文档。
3. 只有在本索引证据不足时，才扩展到相邻分类目录。
4. 不要先全量递归扫描 `references/`。
5. 读取命中文档时优先读和当前问题最相关的片段，不必整库通读。

## 推荐检索顺序

### A. 用户问“怎么配置/怎么实现”
优先查：
1. `references/QUICK_INDEX.md`
2. `references/forms/...` 命中文档
3. 如还不够，再看相邻 `troubleshooting/...`

### B. 用户问“为什么异常/为什么不生效/帮我排查”
优先查：
1. `references/QUICK_INDEX.md`
2. `references/TROUBLESHOOTING_PLAYBOOK.md`
3. `references/forms/...` 命中文档
4. `references/troubleshooting/...` 同类文档
5. 最后再扩展到相邻分类

### C. 用户同时给了现象 + 参数（如 pkBo / pkBoins / 环境）
优先做：
1. 用 H5 页面/接口看真实返回数据
2. 再按本索引找最相关文档验证原因

## 这份索引解决什么问题

- 减少首次排查时的目录扫描开销
- 避免 Windows 控制台中文路径乱码导致的反复探测
- 把用户自然语言问题直接映射到文档路径
- 提高高频问题（审批、提交、权限、同步、接口）的首轮命中率
